The Project:

Create a plan/policy to detect passing three doors by adapting the code from Project 1 ro an iRobot Create 3.

Baysian Network

We used the python library Pomegranate to build and fit our Bayesian network from gathered data. The data we gathered was of the robot heading toward the door with the wall on its left. The network at school proved a challenge when connecting to the robot via ROS. We ended up having to take the robot home and improvised doors to use for training.

Files: See code branch

  • code/
    • bayesian.py - Used to create the bayesian network. See Dockerfile.
    • create3_coverage/ - This is a workspace for the original example code provided by the Create team that we adapted into our data-gathering code.
    • create3_examples_ws/ - This is a workspace for our data collection code adapted from the Create examples provided by the Create team.
    • create3_ws/ - This is a workspace for the Create robot simulator. We used this a few times when developing the code
    • Dockerfile - This is a Dockerfile to use when generating the bayesian network.
    • door_data/ - This is all of our collected data from the robot that we gathered during training.
    • instructions.txt - Basic instructions used when generating the data.
    • pomdp/ - Contains an example run of the POMDP Python code we made with pictures showing the different states and how the probability of the POMDP moved over time. The data.txt file contains the data from the execution of the POMDP and shows the hardcoded observations we provided. The PODMP CPT tables were based on hardcoded values as well.
    • python/ - Contains the source code for generating the distance from the wall as a probability distribution (dist_from_wall.py), the input santitation that distance from wall script uses (ir_intensity.sh) and the trained bayesian network and live_model.py used for live testing. Also contains the ROS API commands for rotating and wall following contained in ros.py and the POMDP Python code contained in pomDP.py.
    • ros2_ws/ - This is a workspace that has the specialized irobot_create_msgs which are customized actions, topics, and messages that the robot uses to communicate in ROS.
    • ros_ign/ - This is a workspace that contains code for linking ROS to Gazebo. This is used by the simulator code.
    • workspace/ - This is a workspace containing Gazebo support code. This is used by the simulator code.
  • model_50n_5r.pxl - model data.
  • Model_50.png - picture represenation of the model.
  • README.md - overall readme
  • style.css - style sheet for this page
  • Tables.csv - final CPT tables
